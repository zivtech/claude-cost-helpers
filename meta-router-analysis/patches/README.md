# Patches

Ready-to-apply patch series for the two public meta-skill bundles measured in `../benchmark/results.md`. Each patch bumps its bundle closer to the router-v2 pattern described in `../improvements/router-v2-spec.md`.

These live here because the MCP tools available to the Claude Code session that generated them are scoped to `zivtech/claude-cost-helpers` and cannot open PRs against other repos. Applying the patches and opening the PRs is a human step.

## Patches

| Bundle | Patch | Change | Turn-10 parent context (Δ) |
|---|---|---|---|
| `zivtech/a11y-meta-skills` | `a11y-meta-skills/0001-*.patch` | Full router-v2 refactor. SKILL.md files shrink to frontmatter stubs; protocol bodies move into `.claude/agents/`. Adds new agent files for `a11y-test` and `perspective-audit`. | 42,149 → 1,465 (**-96.5%**) |
| `zivtech/drupal-meta-skills` | `drupal-meta-skills/0001-*.patch` | Surgical trim. Removes `Companion_Skills`, `Tool_Usage`, `References`, `Final_Checklist`, and similar implementation-detail sections from every SKILL.md. Agents are unchanged. | 9,307 → 6,391 (**-31%**) |

drupal-meta-skills already implements the router-to-agent pattern, so its patch is conservative — the big win was in the existing architecture. a11y-meta-skills keeps its agents on disk but duplicates the full protocol inline in SKILL.md, so its patch is the structural refactor.

## Applying

Both patches are generated by `git format-patch`. Apply with `git am`.

### a11y-meta-skills

```bash
git clone https://github.com/zivtech/a11y-meta-skills.git
cd a11y-meta-skills
git checkout -b router-v2-refactor
git am /path/to/claude-cost-helpers/meta-router-analysis/patches/a11y-meta-skills/0001-*.patch
git push -u origin router-v2-refactor
# Open PR at https://github.com/zivtech/a11y-meta-skills/pulls
```

### drupal-meta-skills

```bash
git clone https://github.com/zivtech/drupal-meta-skills.git
cd drupal-meta-skills
git checkout -b router-v2-trim
git am /path/to/claude-cost-helpers/meta-router-analysis/patches/drupal-meta-skills/0001-*.patch
git push -u origin router-v2-trim
# Open PR at https://github.com/zivtech/drupal-meta-skills/pulls
```

If `git am` fails because the patches were authored with a different author/signing config than your local git, use `--committer-date-is-author-date` or re-author with `--reset-author`. The commits were intentionally generated with `commit.gpgsign=false` (the sandbox that produced them could not reach its signing server for files outside the primary repo) — you can re-sign with `git commit --amend -S --no-edit` after the `git am`.

## Suggested PR descriptions

For a11y-meta-skills:

> **Router-v2 refactor** — adopt the stubs-first pattern. Each SKILL.md shrinks to a frontmatter-delegation stub; protocol bodies move into `.claude/agents/`. Measurement (from [claude-cost-helpers/meta-router-analysis](https://github.com/zivtech/claude-cost-helpers/tree/main/meta-router-analysis)): turn-10 parent context drops from 42,149 → 1,465 tokens (-96.5%), cold-cache volatile tokens per 5-min TTL expiry drop from 41,649 → 965 (-98%). No accessibility content was removed — it moved verbatim from SKILL.md to agents.

For drupal-meta-skills:

> **Trim SKILL.md files** — remove sections that duplicate agent work. Keeps the router-level content (routing table, Use_When, Do_Not_Use_When, high-level Steps); removes Companion_Skills, Tool_Usage, References, Final_Checklist and similar sections that belong in the agent protocol. Measurement (from [claude-cost-helpers/meta-router-analysis](https://github.com/zivtech/claude-cost-helpers/tree/main/meta-router-analysis)): SKILL.md total shrinks from 8,807 → 5,894 tokens (-33%); turn-10 parent context drops from 9,307 → 6,391 (-31%). Agent files are unchanged; the routing contract is preserved.

## Verifying before you push

Each patch includes its full measurement summary in the commit message. To confirm nothing is broken:

```bash
# After git am, from inside the bundle:
ls -la .claude/skills/*/SKILL.md .claude/agents/
# The SKILL.md files should be smaller; the .claude/agents/ directory
# should still contain everything it did before (plus two new files for a11y).

# Optional: run the benchmark against your local branch
python3 /path/to/claude-cost-helpers/meta-router-analysis/benchmark/count_tokens.py \
  .claude/skills/*/SKILL.md .claude/agents/*.md
```

The patches are small enough to review by eye before pushing — the a11y patch is 3,294 lines (mostly removing protocol from SKILL.md and adding it to agents); the drupal patch is 390 lines (section removals only).
